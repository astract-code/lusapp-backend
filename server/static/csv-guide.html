Lusapp – CSV Race Import Guide

Step-by-step rules for preparing your CSV file so every race imports correctly into the app.

Critical – Read First If sport_category, sport_subtype, or date contain any error, that entire row is silently skipped and never appears in the app. There is no error shown in the app — the race simply does not get imported. Follow the rules below exactly.

1. Column Names

Your CSV file must have these exact column names in the first row (header row). Capitalisation matters.

Column Name Required? Description
name REQUIRED Full name of the race or event
sport REQUIRED Same value as sport_category — write the same word in both columns
sport_category REQUIRED The sport group. Must be one of exactly 6 values — see Section 2
sport_subtype REQUIRED The specific type within the sport. Must match exactly — see Section 2
city REQUIRED City where the race takes place (used for search and filtering)
country REQUIRED Country where the race takes place (used for filtering)
continent REQUIRED Continent — must be one of 6 exact values — see Section 3
date REQUIRED Race date in YYYY-MM-DD format — see Section 4
start_time optional Start time in HH:MM format, e.g. 08:00. Leave blank if unknown.
distance optional Free text distance, e.g. 42.2km or 10 miles. The app automatically shows standard distances for known race types, so you can leave this blank.
description optional Short description of the race. Keep it under a few sentences.
participants optional Expected number of participants. Must be a whole number with no commas or symbols, e.g. 5000.
Note on column order The order of columns does not matter — the system reads by column name. But all column names must be spelled exactly as shown above, with underscores where shown.

4. Date Format

The date column must follow the YYYY-MM-DD format exactly. Any other format causes the row to be skipped.

✓ Correct
2025-12-25
2026-04-05
2026-01-01
✗ Wrong — row will be skipped
25/12/2025
December 25, 2025
25-12-2025

Always use four digits for the year, two digits for the month, and two digits for the day. January = 01, not 1.

5. Duplicate Detection

The system automatically skips any row where a race with the same name (case-insensitive) and the same date already exists in the database. This prevents accidental double imports.

If a race was previously imported and you need to update it (change city, description, etc.), you must edit it directly in the admin panel — you cannot update existing races by re-uploading the CSV.

6. After Import

Races imported via CSV are automatically approved They appear immediately in the app for all users. No admin approval step is needed after a CSV upload.

8. Pre-Upload Checklist

Before uploading your CSV, go through this list:

  • Column header names are spelled exactly as listed in Section 1 (no spaces, underscores where shown)
  • Every row has a value in name, sport_category, sport_subtype, and date
  • sport_category is one of: Running, Triathlon, Cycling, Fitness, Swimming, Custom
  • sport_subtype matches exactly one of the subtypes for that category (copy-paste from Section 2)
  • sport contains the same value as sport_category
  • continent is one of the six values from Section 3
  • All dates are in YYYY-MM-DD format
  • No race name + date combination already exists in the database
  • The file is saved as plain CSV (not Excel .xlsx) with UTF-8 encoding
  • Numbers in participants have no commas (e.g. 5000, not 5,000)
  • Text fields containing commas are wrapped in double quotes (most spreadsheet apps do this automatically)
Saving from Excel or Google Sheets In Excel: File → Save As → CSV UTF-8 (Comma delimited) (.csv)
In Google Sheets: File → Download → Comma-separated values (.csv)

9. Why Was My Row Skipped?

After uploading, the admin panel shows how many rows were imported and how many were skipped. If rows were skipped, the most likely reasons are:

  • Wrong sport_category — Does not exactly match one of: Running, Triathlon, Cycling, Fitness, Swimming, Custom
  • Wrong sport_subtype — Does not exactly match a valid subtype for that category (see Section 2)
  • Wrong date format — Date is not in YYYY-MM-DD format
  • Missing name or date — One of those required fields is empty
  • Duplicate — A race with the same name and date already exists in the database
  • Extra spaces — A value like " Running" (with a leading space) will fail. Make sure there are no accidental spaces before or after values.
